Image Preprocessing Pipeline

The image preprocessing pipeline is a highly optimized engine designed to ingest satellite imagery, perform multi-threaded geospatial operations, and execute batched ship detection inference using PyTorch and ONNX Runtime. The pipeline eliminates storage bottlenecks by relying almost entirely on a high-speed RAM disk for intermediate operations.

Pipeline Components & Optimizations

  • Mounted RAM Disk (tmpfs): A virtual memory partition that serves as a zero-latency staging ground for all intermediate outputs (extracted archives, temporary crops, SHP files). This completely mitigates NVMe/SSD read/write bottlenecks during heavy I/O operations.
  • Archive Ingest & Parallel Extract: Handles incoming .zip, .tar, and .tgz bundles. Extraction is offloaded to highly parallelized tools like pigz and unzip, which utilize all available cores.
  • Pipeline Router: Scans the extracted payload and determines whether it should undergo the standard gdalwarp clipping process (if SHP files are present) or jump straight to inference (Image-Only pipeline).
  • GeoPandas SHP Processing: Buffers and reprojects the detection shapes purely in-memory (EPSG:4326), avoiding unnecessary intermediate file writes.
  • Multi-threaded Raster Clipping: Launches highly parallelized gdalwarp jobs via a ThreadPoolExecutor, dynamically allocating all available CPU threads to clip multi-band TIFFs around the buffered areas.
  • Producer/Consumer Decoupling:
    • Producer (CPU): Tiles large TIFFs (without loading them fully into RAM) and normalizes pixel values via GPU tensor stretches (2nd-98th percentile), outputting inference-ready JPGs.
    • Consumer (GPU): A dedicated queue pulls the staged images, loading them into pinned memory for zero-copy DMA transfer to the VRAM. ONNX Runtime then executes batched inference using CUDAExecutionProvider, auto-sizing the batch to avoid cuDNN limits.
  • Metadata Fusion: Merges geospatial coordinates (extracted from SHP or KML/KMZ files) with the ONNX classification predictions into a final Master CSV file.
